# VEGEX — лендинг экспортёра чеснока

B2B-лендинг VEGEX (экспорт премиального высокогорного чеснока из Кыргызстана)
для покупателей в Германии и Польше. Статический сайт, деплой на Cloudflare Pages.

## Архитектура

```
content/
  ru.yaml, en.yaml      единственный источник текстов и данных
templates/
  base.html             каркас страницы (head, шапка, подвал, порядок секций)
  sections/*.html       hero, facts, about, terroir, products, specs (+упаковка),
                        quality, logistics, partners, advantages, contact
static/
  css/main.css          зафиксированная визуальная система (v3, не менять без запроса)
  js/main.js            ванильный JS: reveal-анимации, калибр-бары, меню, форма
  fonts/                исходники шрифтов (Montserrat .woff2) для инлайна
  images/               исходники фото (в git; base64 генерируется при сборке)
build.py                рендер обоих языков → самодостаточные dist/<lang>/index.html
functions/contact.js    Cloudflare Pages Function — обработка формы
```

## Языки

Сейчас в сборке активен только русский (`LANGS = ["ru"]` в `build.py`) — по решению
клиента запускаем русскую версию первой. Переключатель RU/EN в шапке скрывается
автоматически, когда активен один язык.

Когда будет готов согласованный английский контент (сейчас есть черновик клиента,
`content/en.yaml` — на вычитку/актуализацию под финальную структуру v3):

```python
LANGS = ["ru", "en"]
```

и пересобрать — переключатель появится сам.

## Шрифты

Montserrat вендорится в `static/fonts/` (сабсеты latin + cyrillic, веса 400–900,
формат woff2, лицензия SIL OFL) и инлайнится в `<style>` как base64 `@font-face`
с `unicode-range`. Это сделано вместо подключения Google Fonts по двум причинам:
самодостаточность (правило проекта — `dist/*.html` открывается из файла без сети)
и приватность аудитории ЕС (динамическая загрузка шрифтов с гугл-серверов передаёт
IP пользователя третьей стороне). Если `static/fonts/*.woff2` отсутствуют,
`build.py` автоматически откатывается на `<link>` Google Fonts.

## Сборка

```bash
pip install -r requirements.txt
python build.py
```

Результат — `dist/ru/index.html` (и `dist/en/index.html`, когда язык включён):
валидный самодостаточный HTML без внешних зависимостей (CSS/JS/шрифты/фото
встроены). Открывается прямо из файла. `dist/index.html` — редирект на язык
по умолчанию (для Cloudflare Pages).

Локальный предпросмотр:

```bash
python -m http.server -d dist 8000
# ru: http://localhost:8000/ru/   en: http://localhost:8000/en/
```

## Правила разработки

1. **Контент и разметка разделены строго.** Меняется текст — правится только
   `content/*.yaml`. Меняется вёрстка — только `templates/`. В шаблонах текст не хардкодить.
2. **Визуальная система зафиксирована (v3, согласована с клиентом).** Зелёно-оранжевые
   градиенты подчёркиваний, тёмно-зелёные пилюли-подписи, круглые бейджи, карточки
   в стиле fruitexim. Без явного запроса не менять.
3. **Фото встраиваются как base64 на этапе сборки.** В git — только исходники в
   `static/images/`, конвертация в `build.py`. Готовый base64 не коммитить.
4. **Кириллические имена файлов транслитерируются в латиницу** при сборке
   (известная точка отказа — решена в v3, не откатывать).
5. **Языковой переключатель — только RU/EN** (немецкий убран по решению клиента).
6. **Форма — POST на `functions/contact.js`**, без сторонних клиентских SDK.
   Аудитория ЕС, данные покупателей GDPR-чувствительны: направление данных
   (email/CRM/хранилище) согласуется явно.
