# Исходники контента (справочные, не используются в сборке)

Эти файлы не читаются `build.py` — единственный источник для рендера
это `content/ru.yaml` / `content/en.yaml`. Здесь хранятся исходные
материалы клиента для сверки при будущих правках контента.

- `VEGEX_eng_client_source.docx` — английский текст от клиента (структура
  из одного сорта "Royal Purple Garlic"). Согласованная с клиентом v3-вёрстка
  описывает два сорта (Royal + Ivory), поэтому при переносе в `content/en.yaml`
  нужно адаптировать под структуру v3, а не копировать 1:1.
